# 11 Prompt Debugging

## What it means

Prompt debugging means finding out why a prompt gives bad results and fixing the weak parts.

## When to use it

Use it when the model misunderstands the task, ignores your format, or gives low-quality answers.

## Small example

If the model keeps giving long answers, you add a rule like, "Keep each answer under 50 words."

## Key takeaway

When a prompt fails, make the instruction clearer instead of starting from zero.
